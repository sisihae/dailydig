# Daily Music Discovery Agent

## Workflow

1. Read the spec for the module you're about to build (`docs/specs/`). All specifications live in `docs/specs/`. Start with `docs/specs/README.md` for the full index.
2. Read the corresponding plan doc (`docs/plans/`). All implementation plans live in `docs/plans/`. Start with `docs/plans/README.md` for the full index.
3. follow the plan doc when implementing.

Ask the user if something is vague or ambiguous. Make everything clear and planned before starting implementation.
Commit changes to git. Each commit should be a minimal coherent idea.

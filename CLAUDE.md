# Daily Music Discovery Agent

## Workflow

1. Read the spec for the module you're about to build (`docs/specs/`). Start with `docs/specs/README.md` for the full index. specs include 'what' and 'why'. it's like a design document.
2. Read the corresponding plan doc (`docs/plans/`). Start with `docs/plans/README.md` for the full index. plans include 'how'. plans should be actionable, ready to run.
3. follow the plan doc when implementing.

Ask the user if something is vague or ambiguous. Make everything clear and planned before starting implementation.
Commit changes to git. Each commit should be a minimal coherent idea.
